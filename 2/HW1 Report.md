## Домашнее задание №1 - Отчёт
### Условие - Making The Grade
Очень большой и густонаселенный субъект РФ хотел бы внедрить новую систему для проведения стандартизированного тестирования во всех государственных школах с 2-го по 11-й класс.

Пользователи: более 40 000 учащихся, 2000 учителей, 50 администраторов.

Требования:
- Учащиеся смогут пользоваться приложением только в центрах тестирования по всему субъекту, большинство из которых расположены в школах, но не все.
- Учащиеся должны иметь возможность пройти тест, а результаты в конечном итоге должны быть собраны в одном месте, где будут представлены все результаты тестирования по всему штату (по школам, учителям и учащимся).
- Тесты будут включать в себя вопросы с несколькими вариантами ответа, вопросы с краткими ответами и эссе.
- В системе должна быть предусмотрена система отчетности, позволяющая отслеживать, кто из учащихся прошел тестирование и какие баллы получил.
- Вопросы с краткими ответами и эссе будут оцениваться вручную учителями, которые затем будут вносить оценки в систему.

Дополнительный контекст:
- для внесения изменений в систему хранения оценок учащихся требуется одобрение трех различных государственных ведомств, что обеспечивает безопасность данных.
- Субъект РФ не владеет собственным хостинговым центром, а передает его на аутсорсинг сторонней организации.
- Каждый финансовый год проект должен подтверждать свой бюджет.
- Вы находитесь в России, 2026 год

### Сбор и анализ требований
#### Отсев
Реализуемые требования:
| ID    | Требование                                                   |
|-------|--------------------------------------------------------------|
| R1    | Проведение тестирования в центрах тестирования               |
| R2    | Централизованный сбор результатов по всему субъекту          |
| R3    | Три типа вопросов (множественный выбор, краткий ответ, эссе) |
| R4    | Система отчётности                                           |
| R5    | Ручная проверка эссе и кратких ответов учителями             |
| R6    | Тройное согласование изменений в хранилище оценок            |
| R7    | Хостинг на аутсорсе (нет собственного ЦОД)                   |
| R8    | Ежегодное подтверждение бюджета                              |

Матрица рисков по требованиям:
| ID    | Риск  | Вероятность | Влияние | Мера управления |
|-------|-------|-------------|---------|-----------------|
| RSK1  | Нестабильный интернет в центре тестирования | HIGH | CRIT | Офлайн режим тестирования с последующей синхронизацией |
| RSK2  | Учителя не успевают проверить эссе в срок | HIGH | MEDIUM | Выставление сроков проверки и перераспределение заданий между учителями
| RSK3  | Одновременная сдача теста всеми учащимися | MEDIUM | CRIT | Возможность временного масштабирования системы, отложенная обработка результатов тестирования |
| RSK4  | Бюджет не утверждён на следующий год | MEDIUM | HIGH | Модульная архитектура сервиса и возможность его «заморозки» без потери данных |

Acceptance Criteria:
| ID    | Критерий                                                                             |
|-------|--------------------------------------------------------------------------------------|
| AC1   | Учащийся может начать и завершить тест только из авторизованного центра тестирования |
| AC2   | Результаты тестов с множественным выбором оцениваются автоматически после сдачи      |
| AC3   | Учитель может просмотреть и оценить эссе/краткие ответы через веб-интерфейс          |
| AC4   | Администратор может сформировать отчёт по школе/учителю/ученику за указанный период  |
| AC5   | Изменение итоговых оценок возможно только после одобрения представителей 3 ведомств  |
